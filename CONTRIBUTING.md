# 🪻 Contributing

## 🐙 How to Contribute
1. **Fork the repo** and clone your fork.
2. **Create a new branch** for your feature, fix, or improvement.
3. **Make your changes** and write clear, descriptive commit messages.
4. **Test your changes** to ensure they do not break existing functionality.
5. **Open a Pull Request** with a clear summary of what you changed and why.

## 🪭 Code Style
- Follow idiomatic Go practices.
- Keep the code modular.
- Prioritize readability and simplicity.

## ⚡ Issue Reporting
- Check existing issues before opening a new one.
- Include clear steps to reproduce bugs.
- Suggest possible solutions if you have any ideas.

## 🧄 Pull Request Guidelines
- Keep PRs focused and small (one feature or fix per PR).
- Reference related issues (e.g. `Fixes #12`).
- Ensure your branch is up to date with the main branch.
